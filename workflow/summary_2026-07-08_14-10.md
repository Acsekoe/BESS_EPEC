# Workflow Summary - 2026-07-08

## Current Objective

Build a spot-market-only BESS EPEC extension. The project now completely ignores aFRR markets and focuses on strategic investor competition for scarce nodal BESS power access and heterogeneous battery operation constraints.

## Key Decisions

- Removed aFRR, reserve activation, stochastic scenarios, and reserve-market clearing from the active modeling direction.
- Current lower-level model is a deterministic spot-market DC-OPF with fixed BESS capacities.
- Investor layer is not implemented yet. `X_power` and `X_energy` are currently exogenous inputs from processed data.
- Main EPEC challenge: avoid artificial first-mover capture of nodal BESS limits in Gauss-Seidel. Need to compare or design alternatives such as Gauss-Jacobi, damping/projection, or shared nodal-access equilibrium constraints.

## Data Workflow

- `prepare_data.py` reads the cleaned Excel input and writes spot-only processed data.
- `data/input/bess_epec_inputs.xlsx` was cleaned of reserve/scenario/aFRR sheets and columns.
- `data/processed/market_data.json` is the baseline processed model input.
- `data/processed/market_data_experiment.json` is a safe experiment copy.
- PTDF values are calculated in `prepare_data.py` via `calculate_ptdf(...)` and written to both `market_data.json` and `data/processed/ptdf.csv`.

## Model Status

- `primal_market_clearing_model.py` solves only the lower-level primal spot-market clearing LP.
- It prints:
  - objective value;
  - system-wide lambda from `sum_n NetInjection[n,t] == 0`;
  - nodal balance duals;
  - hourly dispatch, charge, discharge, and sanity checks.
- Baseline prices are mostly `70.0877` and `80.0000` EUR/MWh.
- `80` is driven by the peak generator marginal cost.
- `70.0877 = 0.936^2 * 80`, from storage arbitrage with round-trip losses.

## Experiment Status

- `market_data_experiment.json` has N4 demand multiplied by `1.2`.
- This creates nodal price separation without load shedding.
