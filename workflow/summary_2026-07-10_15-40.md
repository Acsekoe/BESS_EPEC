# Workflow Summary - 2026-07-10 15:40

## Current Objective

Prototype a two-investor auction-mediated BESS capacity game before building the full four-investor thesis model.

## Implemented Files

- `auction.py`
  - Defines `Bid`, `AcceptedBid`, and `AuctionResult`.
  - Implements sealed pay-as-bid nodal clearing.
  - Bids are grouped by node and ranked by price.
  - Partial acceptance is supported.
  - Equal-price ties are rationed pro-rata.

- `epec_auction_simulation.py`
  - Runs the chain:
    1. solve initial spot-market LP;
    2. generate sealed bids for two investors;
    3. clear nodal BESS capacity auction;
    4. solve spot-market LP with accepted capacities;
    5. compute spot revenue, degradation cost, CAPEX, auction payment, and profit;
    6. write CSV history.
  - Supports two bidder modes:
    - `--bidder heuristic`: cheap rule-based bids for testing the full iterative chain;
    - `--bidder mpec`: investor bid models using primal-dual strong-duality lower-level representation.
  - Default is currently one MPEC-bid iteration because multi-iteration MPEC results are not stable yet.

- `investor_value_model.py`
  - Adds a single-investor bid model.
  - Lower-level spot market optimality is represented with:
    - primal feasibility;
    - explicit dual feasibility;
    - strong-duality equality.
  - No complementarity/KKT pairs are used.
  - Uses Ipopt for the resulting nonlinear/bilinear model.
  - Current Ipopt settings:
    - `max_iter = 1500`
    - `tol = 1e-4`
    - `acceptable_tol = 1e-3`
    - `linear_solver = ma97`
    - `max_cpu_time = 30`
    - `print_level = 0`

## Interpretation

The software chain now works end to end, including multi-iteration MPEC bidding, auction clearing, spot-market LP solving, and CSV reporting.

However, the current MPEC bidder is not economically reliable. Iteration 2 produced unrealistically high realized spot revenues of roughly 1.2 million EUR/day, followed by a return to negative profits in iteration 3. This indicates instability and/or degeneracy in the optimistic primal-dual strong-duality investor formulation.

Likely causes:

- the investor can choose favorable lower-level primal/dual solutions inside the strong-duality system;
- lower-level LP degeneracy can create optimistic prices/dispatch for the upper-level investor;
- there is no damping or smoothing of bids/accepted capacities;
- auction payments are capped and may not discipline the MPEC bidder enough;
- no explicit regularization or tie-breaking exists in the lower-level optimality representation.

## Next Steps

1. Stabilize the investor value model before interpreting results.
2. Add damping between old and new accepted capacities or bids.
3. Add stronger economic bounds/regularization to prevent unrealistic price/revenue outcomes.
4. Consider fixing or regularizing lower-level dispatch tie-breaking to avoid optimistic dual degeneracy.
5. Compare the current auction-mediated approach against a simpler proportional-allocation Gauss-Jacobi prototype.
6. Only after the two-investor case is stable, add the wind-heavy and solar-heavy portfolio investors.
