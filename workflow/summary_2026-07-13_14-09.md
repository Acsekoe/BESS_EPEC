## Current MPEC Result and Issue

Base single-investor MPEC result:

- profit: about `3.97 million EUR/day`;
- spot revenue: about `3.99 million EUR/day`;
- degradation cost: about `15,007 EUR/day`;
- CAPEX daily: about `3,385 EUR/day`;
- load shedding remains about `200.68 MWh/day`;
- total storage throughput is about `2,001 MWh/day`, roughly `2` equivalent full cycles/day.

The huge profit is not normal spot arbitrage. It is caused by network-constrained load shedding and VOLL pricing:

- LMPs hit `10,000 EUR/MWh` in scarcity hours.
- Load shedding occurs mainly at `N2` in hours `7-9` and `N4` in hours `19-21`.
- Binding lines are mainly `L12` and `L45`.
- The investor installs BESS and discharges into scarcity nodes/hours while load shedding remains marginal, so it earns VOLL prices.

Standalone primal LLP with the MPEC capacities gives almost the same lower-level objective, but materially different investor revenue. This indicates lower-level degeneracy / optimistic primal-dual selection, especially in scarcity-price hours.

## Sensitivity Tests

- A tiny BESS dispatch cost in the LLP objective was tested and then reverted.
  - It preserved primal-dual consistency.
  - It did not fix the revenue-selection gap.
  - Conclusion: the main issue is not unnecessary BESS cycling, but scarcity/dual-price degeneracy.
- Doubling all line limits was tested and then removed from the code.
  - It eliminated load shedding and reduced max LMP to `80 EUR/MWh`.
  - Profit dropped to about `13,762 EUR/day`.
  - However, it also removed congestion rents, so it is too blunt as a final benchmark.
- Targeted line-limit search found a narrow no-load-shedding but congested no-storage baseline:
  - examples: `L12 = 480 MW`, `L45 >= 310 MW`, or `L12 = 485 MW`;
  - no load shedding;
  - LMP spread around `779 EUR/MWh`;
  - `L12` remains binding around hour `8`.

## Current Interpretation

The reduced MPEC works mathematically. The open modeling issue is benchmark calibration:

- We need a base case with no load shedding before BESS investment.
- We still want congestion-driven LMP differences so the BESS can earn congestion/arbitrage rents.
- The current original line limits are too tight and create VOLL scarcity.
- Doubling all line limits is too loose and removes congestion.
