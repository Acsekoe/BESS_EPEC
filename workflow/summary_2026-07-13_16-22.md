## DECISION: Treat Dual Degeneracy via Reference-Price Settlement

Question considered: "should we pick the dual solution with minimal LMPs?" Answer: **no**.

- Min-LMP is not neutral (pessimistic for discharging, optimistic for charging hours), not a single objective (needs arbitrary scalarization), and imposing it inside the MPEC is a lexicographic bilevel-in-bilevel.
- Data perturbation (RHS jitter, the textbook cure for dual multiplicity) does NOT protect here: `X_power` sits on lower-level RHS, so the strategic investor can re-engineer degeneracy wherever it pays.

Adopted approach (standard for strong-duality MPECs in the storage-investment literature):

1. MPEC internal lambda = the investor's *belief*; keep the formulation unchanged for optimization.
2. **Post-solve settlement step**: fix `X_power`/`X_energy`, solve the standalone LLP with HiGHS (deterministic basic duals = reference prices), recompute revenue/degradation/profit at those prices. Report `profit_at_reference_prices` alongside the optimistic MPEC profit in `summary.json` and the console. The gap = degeneracy exposure.
3. Optionally report the pessimistic bound (probe MIN) so each result carries an interval; stable investment across the interval = trustworthy equilibrium.
4. **In the EPEC loop**: one reference LLP solve per iteration settles ALL investors at the same prices (fixes cross-investor price inconsistency) and drives convergence checks.

Caveat: settlement changes reporting, not the optimizer's incentive - the MPEC still *chooses* X believing optimistic prices. If reference-settled profit goes negative for the chosen X, the equilibrium is an artifact; the clean structural fix then is strictly concave (QP) demand, which makes lower-level prices unique so belief and settlement coincide. Keep as fallback.
