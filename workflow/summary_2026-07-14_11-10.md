# Workflow Summary - 2026-07-14 11:10

## Current Objective

Stabilize the single-investor MPEC before rebuilding the multi-investor EPEC by making lower-level dual-price degeneracy explicit and adding a deterministic dual-selection settlement rule.

## Current Model Workflow

`model/single_investor_mpec.py` solves the investor problem first. The investor chooses nodal BESS power and energy capacities while embedding the lower-level spot-market LP through primal feasibility, dual feasibility, and strong duality.

The embedded MPEC objective uses the MPEC's own selected `lambda[n,t]`, so the first reported profit remains the **MPEC optimistic profit**. If the lower-level LP has multiple valid dual/LMP solutions, the MPEC can select the one most favorable to the investor.

After the MPEC solve, capacities are fixed and the script runs settlement diagnostics:

1. **Reference LLP settlement**
   - Fixes `X_power` and `X_energy`.
   - Solves the standalone primal LLP with HiGHS.
   - Computes profit using HiGHS-returned nodal-balance duals.
   - This is deterministic solver settlement, but not guaranteed to be economically unique if the LP is degenerate.

2. **Minimum-norm dual selection rule**
   - Builds the lower-level dual model at the fixed MPEC capacities.
   - Restricts the dual model to the optimal dual face with:
     `dual objective = reference LLP objective`, within `--dual-face-tolerance-eur`.
   - Selects one valid price vector by solving:
     `min sum(lambda[n,t]^2)`.
   - Recomputes investor profit at those selected LMPs using the reference LLP dispatch.
   - This gives an explicit settlement rule: choose the least-extreme admissible LMP vector.

3. **Dual-face revenue bounds**
   - At fixed reference LLP dispatch, solves LP probes over the same optimal dual face.
   - Reports pessimistic and optimistic investor revenue/profit bounds.
   - These are price-degeneracy bounds, not full joint primal-dispatch/dual degeneracy bounds.

## Important Decision

- `market_data.json` follows the clean benchmark / fallback behavior.
- `market_data_experiment.json` contains the recent hand-maintained tiered-demand experiment data.

The active degeneracy handling is now based on **post-solve dual selection**, especially the minimum-norm `lambda` rule, not on replacing demand with a synthetic runtime linear curve.

## Current Profit Concepts

Each run can now distinguish:

- `profit_eur_per_day`: optimistic embedded MPEC profit.
- `profit_at_reference_prices_eur_per_day`: standalone primal LLP settlement using HiGHS duals.
- `profit_at_min_norm_dual_prices_eur_per_day`: settlement using the minimum-`|lambda|^2` dual rule.
- `pessimistic_profit_eur_per_day` / `dual_face_optimistic_profit_eur_per_day`: price-degeneracy bounds over admissible optimal duals.

For the experiment-data 50 MW existing-fleet test:

Interpretation: the minimum-norm rule reduces the optimistic MPEC artifact but is still more favorable to the investor than the HiGHS reference duals in this case.
